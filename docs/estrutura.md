# Estrutura inicial

Esta estrutura mantém o protótipo funcionando, mas separa responsabilidades.

## Próximas divisões recomendadas

1. Remover dados mockados de `database.js`.
2. Criar camada de serviço para Supabase.
3. Criar arquivos separados para políticas de permissão.
4. Migrar templates HTML para componentes, caso o projeto vá para React, Vue ou outro framework.
5. Criar uma pasta `assets/vendor` somente se decidir baixar bibliotecas em vez de usar CDN.
